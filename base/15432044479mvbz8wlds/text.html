В таком случае начните проект сначала. И желательно не в PyCharm, а с консоли( или терминала ). Там весь процесс виден как на ладони. Это в PyCharm просто проект создали и все готово. А если научитесь делать через консоль-терминал, тогда сразу поймете, что не так. Процесс такой( у меня во всяком случае, пишу на примере mac os):


Все в терминале:

1. Создаете папку для виртуального окружения  - $ mkdir "имя папки"

2. Заходите в папку - $ cd "имя папки"

3. Прописываете $ virtualenv .(точка обязательна), если не установлено ->> Установить sudo pip install virtualenv

4. Активируете созданное виртуальное окружение - $ source bin/activate

5. Устанавливаете django - $ pip install --upgrade django

6. Создаете проект - django-admin.py startproject 'имя проекта'

7. Переходите в папку с проектом, так как там лежит manage.py - cd 'имя проекта'

Дальше можно делать миграции

8. Прописываете; python manage.py makemigrations, python manage.py migrate, python manage.py createsuperuser

Можно запускать сервер: python manage.py runserver

9. Ну и создаете приложение: python manage.py startapp 'имя приложения'


Погуглите, если работаете на виндоус, там слегка отличаются пути и команды. Но в целом процесс тот же.
